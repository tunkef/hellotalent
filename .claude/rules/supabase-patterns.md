# Supabase Patterns

## Query Patterns
- Always use `.maybeSingle()` instead of `.single()` for queries that might return no rows (new users)
- Use `get_my_candidate_id()` in RLS policies for candidate-owned tables
- Use `is_employer()` helper for employer read access checks
- Child table RLS: always check parent candidate is_active + profile_completed

## Identity Column Types (CRITICAL)
- candidates.id = bigint (GENERATED BY DEFAULT)
- companies.id = bigint
- brands.id = bigint
- hr_profiles.id = uuid (FK to auth.users)
- All child tables (experiences, education, etc.): id = uuid, candidate_id = bigint

## Upsert Rules
- GENERATED ALWAYS columns silently reject upserts — verify column type first
- Always specify onConflict column: `.upsert(data, { onConflict: 'id' })`
- Requires UNIQUE INDEX on the target column

## Storage
- Bucket: 'cvs'
- Avatar path: `avatars/{user_id}.{ext}`
- CV path: `{user_id}/{timestamp}_{filename}`

## Auth
- Candidate login → profil.html
- Employer login → ik.html
- Cross-role prevention in giris.html
- user_metadata.role === 'employer' for routing

## RLS Policy Naming
- Candidate CRUD: `{table}_select_own`, `{table}_insert_own`, etc.
- Employer read: `{table}_employer_read`
